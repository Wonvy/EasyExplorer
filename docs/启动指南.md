## 📱 应用界面说明

### 左侧边栏
- **磁盘驱动器** - 显示所有可用的磁盘驱动器（C:\、D:\ 等）
- **收藏夹** - 您收藏的文件夹快速访问
- **快速访问** - 常用文件夹

### 中间主区域
- **工具栏** - 后退/前进、排序、视图切换按钮
- **路径栏** - 当前路径，点击可快速跳转
- **文件列表** - 显示当前目录的文件和文件夹

### 底部状态栏
- 显示当前目录的统计信息（文件数、文件夹数、总大小）

## 🎮 基本操作

### 文件浏览
- **单击** 磁盘驱动器 → 浏览该驱动器
- **双击** 文件夹 → 进入文件夹
- **双击** 文件 → 用默认程序打开

### 导航
- **后退按钮** (←) - 返回上一个位置
- **前进按钮** (→) - 前往下一个位置  
- **上一级按钮** (↑) - 前往父文件夹
- **路径栏** - 点击任意部分快速跳转

### 排序
- **按名称** - 字母顺序排序
- **按日期** - 创建时间排序
- **按修改日期** - 最后修改时间排序
- **按类型** - 文件扩展名排序

### 视图模式
- **列表视图** (☰) - 紧凑列表
- **图标视图** (▦) - 大图标显示
- **分组视图** (⊞) - 按类型分组
- **时间轴视图** (≡) - 按时间线显示

### 测试 Tauri 命令

在浏览器控制台中可以直接测试 Rust 后端命令：

```javascript
// 获取驱动器列表
await window.__TAURI__.tauri.invoke('get_drives')

// 读取目录
await window.__TAURI__.tauri.invoke('read_directory', { path: 'C:\\' })

// 检查路径是否存在
await window.__TAURI__.tauri.invoke('path_exists', { path: 'C:\\Windows' })
```

## ⚠️ 已知限制

### 当前版本的限制：

1. **文件图标** - 使用 Emoji 图标，不是真实的文件图标
2. **右键菜单** - 尚未完全实现
3. **文件搜索** - 待实现
4. **文件预览** - 部分功能待实现
5. **拖拽操作** - 待实现

### 为什么？

这些功能依赖 Node.js 原生模块，需要在 Rust 端重新实现。
基础文件浏览功能已经完全可用！

## 🐛 遇到问题？

### 常见问题

#### 1. 界面空白
- 按 `F12` 查看控制台错误
- 确认使用 `npm run dev` 启动

#### 2. 驱动器不显示
- 查看控制台是否有错误
- 确认 Rust 后端正常运行

#### 3. 无法打开文件
- 确认文件权限
- 查看控制台错误信息

### 获取帮助

- 查看 [`故障排除.md`](故障排除.md) - 详细的故障排除指南
